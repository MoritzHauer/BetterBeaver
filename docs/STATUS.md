## Works today

Offline-first PWA; content (Kyrgyz + a beaver-themed "Meet BetterBeaver" Introduction topic, replacing the old exercise-type-showcase demo content 2026-07-20) served from the Supabase backend through an IndexedDB cache with strictly opt-in updates, the bundled tree demoted to a frozen first-run seed/fallback, everything validated on every path (plan 0012 steps 1–2); signed-in maintainers edit and publish their documents in-app (magic-link auth, autosaved drafts, atomic validated publish); SM-2 review queue, streak, Topic → Lesson → Unit hierarchy with unlock gates at both levels (skippable via confirm) and lesson/topic-level practice shuffle; 10+ task types incl. sentence building, with cloze/build hint buttons; a paginated Unit screen (dot-trail across Overview/Theory/Vocabulary/Concepts/Examples, per-note and chunked-overflow sub-pagers, swipe/keyboard navigation) with a sticky Practice bar that launches one pooled, shuffled session across the whole unit's tasks — task pinning now lives in that session itself, and Lesson/Topic/TopicList screens show graphical progress bars instead of plain text (plan 0010); per-domain vocabulary screen with learner lists, shipped word families, ad-hoc study incl. dynamic 5-pair matching over saved words, TTS read-aloud; tap-to-lookup on post-answer session surfaces, vocabulary rows, and note views (best-effort resolve with add-word fallback), with hand-authored compound breakdowns in the entry popup; learner-created lexicon entries; per-domain review queue built from scheduling units (cloze blanks reviewed individually, grammar notes as self-graded flashcards, pinned tasks first) and streak; JSON export/import backup. All learner state persists in `localStorage`. Visual identity: beaver mascot start screen, app-icon set, SVG UI icons (plan 0009). Auto-deploys to GitHub Pages on every push to `main` (`.github/workflows/deploy.yml`, served under `/BetterBeaver/`).

## Handoff backlog (not yet built)

Prioritized; each implementation entry points at its self-contained spec (`specs/`) or scoping source. Work top-down unless the owner reorders. Items 2–5 are engineering, executable as written via `/delegate`; item 1 is content authoring and runs under `/author` + the `/ingest` skill (content decisions are never delegated — plan 0007 decision 2); 6–7 need an owner decision first.

1. **Manual ingest backlog (13 lessons)** — content is the product; each lesson is one `/ingest` run (the executable checklist is `.claude/skills/ingest/SKILL.md` — outside `docs/` by design, since skills must live under `.claude/` to be invocable; [plan 0007](plans/0007-ingest-kyrgyz-manual.md) is its design record). Source material is the Peace Corps manual in the owner's vault (`~/vault/sources/kyrgyz/`) — **external to this repo**; ingest is blocked without access to it. Readiness is split: **5 lessons scoped and ready to run** (Transportation, Bazaar, Post Office, Appearance, Weather — line ranges in plan 0007 Context); **8 unscoped** (Food, Apartment, Kiosk, Grocery, Department Store, Health, Daily Routine, At Work) — each needs its own line-range/pedagogy scoping pass first, which is design work, not a mechanical step. Per-lesson labor varies (plan 0007 resolved questions). The **Kyrgyz content backlog** rides along: lexicon gaps surfaced by tap-to-lookup misses, and cloze blanks on names/inflected forms to re-author per plan 0008 point 11.
2. **Proposal flow** — non-maintainer edits with maintainer diff review; completes the plan 0012 editing vision. Spec: [specs/0012-proposal-flow.md](specs/0012-proposal-flow.md) (backend tables/RLS already live).
3. **Asset pipeline** — Supabase Storage upload + cached learner delivery; unblocks the audio/picture task-type editors and new asset-backed content. Spec: [specs/0012-asset-pipeline.md](specs/0012-asset-pipeline.md).
4. **Editor long tail** — missing forms (resources, unlock chaining, links), draft preview, in-app topic creation, admin listing, rollback UI (UI-audit W2–W7 done 2026-07-19). Spec: [specs/0012-editor-long-tail.md](specs/0012-editor-long-tail.md) (item 1's asset-backed task editors depend on 3).
5. **Progress sync + minimal settings screen** — opt-in, local-first, pinned merge rules; includes the app's first Settings screen (sync opt-in only). Spec: [specs/0012-progress-sync.md](specs/0012-progress-sync.md).
6. **Audio production decision** — Kyrgyz audio/picture task types ship (plan 0002) but no unit uses them; blocked on TTS vs. recordings, open since plan 0002. Decision needed before recording work; the asset pipeline (3) is the delivery half.
7. **Play Store packaging** — PWA installs from the browser today; TWA/Bubblewrap when a store listing is wanted. Needs a decision to want it; no spec.
