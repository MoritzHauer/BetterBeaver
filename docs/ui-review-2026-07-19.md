# UI review: contrast + visual audit (2026-07-19)

Scope: all screens in `apps/web`, both themes. Ratios computed per WCAG 2.1
relative luminance from the tokens in `apps/web/src/styles.css` (alpha colors
blended over their actual parent). AA thresholds: 4.5:1 normal text, 3:1
large text and non-text UI. Review only — no fixes applied.

## Critical

### C1. Edit screen "Validate & publish" is invisible in dark mode (1.07:1)

`styles.css:178` `.card > button` and `styles.css:140` `button.primary` have
equal specificity (0,1,1); `.card > button` comes later, so any `button.primary`
that is a **direct child of a `.card`** loses its amber fill (`background: none`,
`border: none`, plus `width: 100%`, `text-align: left`).

Affected:

- `EditScreen.tsx:491` — "Validate & publish" inside `<div className="editor-publish card">`.
  Text keeps `--on-primary` `#241503` (dark brown) but sits on `--card-bg`:
  **dark 1.07:1 (unreadable — this is the reported dark-on-dark)**, light 16.2:1
  (readable, but the primary button silently renders as plain left-aligned text).
  "Discard draft" (`EditScreen.tsx:508`) is also restyled full-width/borderless.
- `AuthorScreen.tsx:91` — "Send sign-in link" inside `<form className="card">`,
  same collision, both themes.

The whole-card tap-target rule was written for card _list rows_
(AuthorScreen doc list, lesson/unit cards) and accidentally captures action
buttons inside card-styled panels.

### C2. `--incorrect-fill` reused as text color fails hard in dark mode

`styles.css:892` `.error-text` and `styles.css:896` `button.danger` color
text with `var(--incorrect-fill)`. That token is a **fill** (a background
under white text — `#8e1f1c` in dark, `styles.css:74`), not a text color.
It only works as text in light mode by coincidence (`#c62828` = 5.5:1).

Dark mode ratios:

- error text on `--bg`: **2.08:1** (load errors, "unknown item" messages)
- error text / Delete / Discard buttons on `--card-bg` (publish panel):
  **1.87:1** — publish validation errors are near-invisible exactly when the
  author needs to read them.

Affected surfaces: every `.error-text` and `.plain danger` in EditScreen
(publish error list, Delete item/task/note/entry/family, Discard draft),
AuthorScreen sign-in errors, ErrorScreen if it uses `.error-text`.
A dedicated `--error-text` token (e.g. reuse `--incorrect-border` `#ef5350`,
8.0:1 on dark bg) is the natural shape of a fix.

## Failures (AA)

### F1. `--locked-fg` below AA in both themes

- Light: 3.42:1 on card-bg, 3.69:1 on bg. Dark: 3.69:1 / 4.12:1. Need 4.5:1.
- Locked lesson/unit cards are still _active_ buttons (tap → skip-ahead
  confirm, `LessonScreen.tsx:73`), so the WCAG inactive-component exemption
  does not apply. Also used by `.start-tagline` (StartScreen), which is plain
  content with no exemption argument at all.

### F2. Input/card borders far below non-text contrast (1.4.11)

- `--card-border` vs backgrounds: light 1.43/1.33:1, dark 1.63/1.46:1 (need 3:1).
- For decorative card edges this is a judgment call, but text inputs,
  textareas, and selects (heavily used across the edit screen forms) rely on
  this border as their only boundary — field extents are hard to perceive,
  especially in dark mode.

### F3. Light-mode progress bar fill vs track: 2.49:1

`--primary` fill on `--card-bg` track (`styles.css:297-311`, session header).
Meaning-bearing non-text UI, needs 3:1. Dark passes (6.1:1).

### F4. `.unit-practice-count` 4.22:1 (both themes)

`--on-primary` at 0.75 opacity on `--primary` (`styles.css:661`), 0.85rem
text — needs 4.5:1. Opacity ≥ 0.82 would pass.

### F5. Light-mode `.badge.done` 4.02:1

`--correct-border` on blended `--correct-bg` (`styles.css:431`), 0.8rem
uppercase text. Dark passes (4.72:1).

## Warnings / minor

- **W1. Light-mode focus outline 2.68:1** — `--primary` outline vs `--bg`
  (`styles.css:156`), just under the 3:1 non-text minimum. Dark passes (6.8:1).
- **W2. Trail dots are 8px tap targets** — `UnitScreen.tsx:311-318` renders
  page dots as buttons at `0.5rem` with `min-height: 0`; below even the 24px
  WCAG 2.5.8 minimum and far below the project's own 44px button convention.
  Swipe + prev/next exist as alternates, which softens but doesn't remove it.
- **W3. Editor row actions 36px** — `.editor-row-actions .plain`
  (`styles.css:857`) opts down to 36px height with small padding; ↑/↓/Edit/
  Delete are the densest tap zone in the app. Above WCAG minimum, below the
  repo's 44px norm.
- **W4. Card padding gaps in AuthorScreen** — `.card` has no padding of its
  own; only `.card > div` and `.card > button` add it. `<p className="card">`
  ("Check your email…", "No documents yet…") and `<form className="card">`
  (sign-in) therefore render text flush against the card border (p margins
  fall outside the border).
- **W5. Read-only edit mode looks editable** — when
  `record.schema_version > CONTENT_SCHEMA_VERSION`, `change()` silently
  no-ops (`EditScreen.tsx:387-394`) but inputs carry no `readOnly`/`disabled`
  attribute or styling; typing just doesn't stick. The status line does say
  "read-only", but the fields themselves give no signal.
- **W6. Long doc/entity ids can overflow** — `EditScreen` renders `{docId}`
  as `<h1>` and ids as headings with no `overflow-wrap`; only `.editor-list li`
  has `overflow-wrap: anywhere`.
- **W7. Placeholder text unaudited** — `new-id-in-slug-form` etc. use the
  browser default placeholder color on `--card-bg`; typically ~2.5–3:1.
  Worth an explicit `::placeholder` color if the forms keep placeholders.

## What passes (spot-checked, both themes)

Body text (15.3–16.9:1), card text, primary buttons (6.5:1), `.status` at
0.7 opacity (6.1–7.9:1), accent/eyebrow/streak/stat-value, action-bar
verdict states, session correct/incorrect cards, badges, done-marks (dark),
dark-mode focus outline and progress fill.

## Suggested priority

1. C1 (publish button invisible) and C2 (error text invisible) — both are
   dark-mode blockers in the authoring flow.
2. F1–F3 — token-level fixes in `styles.css`.
3. F4/F5/W1 — one-line token or opacity tweaks.
4. W2–W7 — batch into the next editor polish pass.
